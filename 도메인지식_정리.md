# 풍력발전 기초 도메인 지식 정리

> BARAM 2026 대회 데이터(태백가덕산풍력) 기반 학습 노트

---

## 1. 에너지 생성 원리

```
바람(공기의 운동에너지) → 날개가 받아 회전(기계에너지) → 발전기(전기에너지) → 전력망
```

- **로터(rotor)** = 날개(블레이드 3장) + 중심축(허브). 실제로 회전하는 부분
- 로터 중심(허브)의 높이 = 허브고도. 우리 단지는 117m

## 2. 핵심 공식

```
P = (1/2) × ρ × A × v³ × Cp
```

| 기호 | 의미 | 단위 |
|---|---|---|
| P | 터빈이 만드는 전력 | W |
| ρ (로) | 공기밀도 | kg/㎥ |
| A | 로터가 쓸고 가는 면적 | ㎡ |
| v | 풍속 | m/s |
| Cp | 효율 계수 | 무차원 |

- Cp를 제외한 부분 `(1/2)ρAv³`은 **바람이 가진 전체 에너지**이다
- 전부 뽑을 수는 없다: 전부 뽑으면 로터 뒤 공기 속도가 0이 되어야 하는데, 그러면 공기가 빠져나가지 못해 뒤가 **막히게** 된다
- 이 이론적 한계가 **베츠(Betz) 한계 = 16/27 ≈ 59.3%** (Cp가 절대 넘을 수 없는 상한)
- 실제 상용 터빈은 공기역학·기계 손실까지 빼서 최적 조건에서 **Cp ≈ 40–50%**
- Cp는 상수가 아니라 풍속에 따라 변하는 함수이고, 이것이 파워커브의 곡선 모양을 만든다
- **v³(세제곱)인 이유**: 입자당 에너지(v²) × 초당 통과량(v) = v³. 풍속 2배 = 에너지 8배

## 3. 실제 운전 로직 (파워커브)

| 지점 | 우리 실측 (V126, kWh/10분) | 무슨 일이 |
|---|---|---|
| 컷인 (약 3m/s) | 0–3m/s 구간 출력 0.0 | 바람이 너무 약하면 아예 안 돎. 우리 데이터에서 풍속의 25%가 4.1m/s 미만 — **시간의 약 1/4은 거의 놀고 있음** |
| 상승 구간 (3–13m/s) | 17.8 → 523 | 풍속³ 법칙으로 급상승. 예보 오차에 가장 민감한 구간 |
| 정격 (약 13m/s) | 13–15m/s에서 531로 플랫 | 발전기 한계 도달. 날개 각도(피치)를 틀어 일부러 바람을 흘려보내며 출력 고정 |
| 고풍속 저감 (15–20m/s) | 478로 감소 | 돌풍·진동에서 기계 보호를 위해 출력을 줄임 |
| 컷아웃 (약 22m/s) | 20m/s+ 구간 319로 급락 | 태풍급이면 날개를 세우고 **완전 정지** — "꼬리 절벽" |

- **꼬리 절벽**: 컷아웃 이후 발전량이 낭떠러지처럼 급락하는 부분. 예보가 2m/s만 틀려도 "만발전 vs 0"이 갈리는 전부-아니면-전무 구간
- 이 **S자 + 꼬리 절벽** 모양 때문에 풍속 → 발전량이 **비선형**이고, **예측 오차의 대가(파워커브의 기울기)가 풍속 구간마다 다르다**
  - 무릎 구간(10–13m/s): 기울기 최대 → 오차 대가 최악 (우리 FICR 진단에서 0.283으로 최저)
  - 정격 구간: 기울기 ≈ 0 → 예보가 틀려도 출력은 맞음

## 4. 지형과 배치

- **풍향장미**: 서남서–서 계열 바람이 시간의 57%, **에너지 기여로는 80%** (자주 불 뿐 아니라 불 때 강함 — 평균 9.0m/s vs 동풍 5.1m/s)
- **터빈 배치**: 17기가 남북 능선을 따라 일렬 — 주풍향(서)과 직각으로 세워 앞 터빈이 뒷 터빈의 바람을 가리는 **후류(wake) 손실**을 최소화한 설계
- 같은 단지 안에서도 터빈별 이용률이 23–40%로 갈림 → 능선 위 몇백 미터 차이로 바람 자원이 약 1.7배 차이. 산악 풍력의 특징이자 **격자 평균보다 격자 최대(max) 피처가 강했던 이유**
- **허브고도 117m**: 지면 마찰 때문에 지상 10m 바람보다 상공 바람이 강하고 안정적 → 예보 데이터에서 GFS 100m·850hPa 바람이 유용했던 이유

## 5. 이용률 (Capacity Factor)

- 그룹 평균 이용률 26–33% — 21.6MW 발전소가 연평균으론 6–7MW어치만 생산
- 출력 분포가 강한 우측 치우침 (중앙값 95 ≪ 평균 192): **대부분 시간은 작게 생산하고, 가끔 몰아서 많이 생산**
- 이 비대칭 구조가 발전량 예측이 어려운 근본 원인이자, 평가 산식의 10% 컷(저발전 시간 채점 제외)과 맞물리는 지점

## 6. 요약

> **바람의 에너지는 풍속³에 비례하고, 터빈은 컷인–정격–컷아웃 로직으로 그것을 S자 곡선으로 수확하며, 능선 지형과 풍향이 바람의 질을 결정한다. 우리는 이 사슬 전체를 예보 데이터로 하루 전에 재구성하는 문제를 풀고 있다.**

# 데이터 활용 방향

## 풍향 → 입력 피처

- **왜 필요한가**: 풍속이 완전히 같아도 서풍일 때와 동풍일 때 발전량이 다르다 (실측: 같은 풍속에서 서풍이 동풍보다 약 1.3배 출력 — 동풍은 능선 뒤에서 넘어오며 난류가 심하기 때문). 즉 **풍속만으로는 발전량이 유일하게 정해지지 않으므로**, 풍향까지 함께 넣어야 정확한 예측이 가능하다
- **어떻게 구했나**: 예보의 u/v 성분에서 풍향 각도를 계산(atan2)한 뒤 **sin/cos 두 값으로 변환**해 피처로 입력
  - sin/cos로 쪼개는 이유: 각도 359°와 1°는 사실상 같은 북풍인데 숫자로는 정반대 → 원 위의 좌표로 바꿔야 방향의 연속성이 보존됨
- 베이스라인에는 없던 피처 — 우리가 추가했고, permutation importance 상위권으로 실제 점수에 기여 중

## 풍속 → 입력 피처

- 원본 데이터에는 풍속 컬럼이 없고 **u(동서)·v(남북) 성분**만 있다
- **u/v를 활용해 풍속을 직접 계산** (피타고라스 정리):

```
풍속 ws = √(u² + v²)
```

- 베이스라인은 u/v 격자 평균 원값을 그대로 사용했는데, 두 가지 문제가 있다:
  1. **모델이 피타고라스를 재발견해야 함**: 풍속 정보가 u/v에 숨어 있어 트리 모델이 비효율적으로 학습
  2. **벡터 상쇄**: 격자마다 방향이 다른 바람을 u/v 상태로 평균하면 서로 상쇄되어 풍속이 과소평가됨 (동풍 5m/s + 서풍 5m/s의 u 평균 = 0)
- 우리 방식: **격자별로 풍속을 먼저 계산한 뒤** 평균/최대/표준편차로 집계 — 순서를 바꿔 상쇄를 방지
- 추가로 풍속³(공식의 v³항), 공기밀도(ρ = 기압/(287×기온)), 출력 프록시(ρ·ws³)까지 물리량을 분해해 피처로 입력

---

*범위 표기는 물결표(~) 대신 –를 사용함 (마크다운 취소선 충돌 방지)*
